Automation & NPM Scripts

Audio Generation Commands

# Generate audio for specific languages npm run generate:english # Generate English audio npm run generate:spanish # Generate Spanish audio npm run generate:german # Generate German audio # Force regeneration with new voices npm run generate:english -- --force # Regenerate all English

Translation Management

# Crowdin integration npm run deploy:crowdin-dev # Upload to Crowdin npm run fetch:translations # Get latest CSV from GitHub # Dashboard translations npm run i18n:csv-to-json # Convert CSV to JSON npm run i18n:validate # Check completeness

Deployment Commands

# Deploy to GCP buckets npm run deploy:translations-dev # Deploy all assets to dev npm run deploy:translations-prod # Deploy to production npm run deploy:web # Deploy web dashboard # Validation & testing npm run validate:core-tasks # Test against core-tasks npm run test:surveys # Validate survey functionality

Troubleshooting & Best Practices

Common Issues & Solutions

⚠️

TTS Service Errors

# Check API keys python generate_speech.py English --force
📝

CSV Format Problems

python utilities/fix_csv_formatting.py
☁️

GCS Permission Issues

npm run setup:service-account-dev
🌍

Crowdin Upload Failures

Verify language code standardization (hyphens vs underscores)

Developer Best Practices

Before Deployment

  • Always use dry-run modes
  • Validate translation coverage
  • Test audio generation
  • Check Crowdin project health

During Development

  • Monitor API usage and costs
  • Use force regeneration sparingly
  • Validate across all languages
  • Keep documentation updated

Quality Assurance

  • Run Cypress validation tests
  • Check translation consistency
  • Verify audio quality
  • Test production deployments
